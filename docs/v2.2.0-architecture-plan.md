# Pangu Nebula v2.2.0 — 五大能力整合架构方案

> **目标**: 在单版本迭代中实现 AI 操作电脑的完整能力链 —— 工具调用 + 命令执行 + 代码执行 + GUI 操作 + LanceDB 知识库

**版本**: v2.2.0
**日期**: 2026-07-16
**架构理念**: Phase 0 基础先行 → Phase 1-5 并行实现 → Phase 6 集成验收

---

## 一、现状缺口

| 能力 | 现状 | 缺口 |
|---|---|---|
| LLM 对话 | stream_reply 仅 token/done/error | 缺 tools 协议、缺工具调用循环 |
| RAG 检索 | RAGService mock(TF-IDF) | 需替换 LanceDB、接入对话、多模态解析 |
| Python 沙箱 | PythonSandbox 完整 | 未作为 tool 暴露给 LLM |
| 终端 | TerminalService mock(无 PTY) | 需装 ptyprocess、作为 tool 暴露 |
| Browser/Computer Use | Rust 骨架 is_skeleton=true | 需 Python 侧 Playwright 兜底 |
| 安全 | InjectionGuard 三类检测 | 未接入 stream_reply |
| 数据模型 | Persona/Conversation/Message 基础 | 缺能力开关、tool_calls 持久化 |
| 前端 | ChatPanel 仅 token/error/done | 缺工具卡片 UI、approval 交互 |

---

## 二、总体架构

```
用户消息
  ↓
[stream_reply 入口]
  ↓
 InjectionGuard 安全检查 (general context)
  ↓
 RAG 检索 → 注入知识库上下文到 system prompt
  ↓
 构建 messages + tools schema
  ↓
 Provider.generate(messages, tools=...) ← tools 来自 ToolRegistry
  ↓
 ┌─ 纯文本 → yield token → 完成
 └─ tool_calls → yield tool_call 事件
       ↓
   权限检查 (ApprovalBanner)
       ↓
   执行工具 (web_search / file_read / file_write
            / execute_command / execute_code
            / browser_navigate / screenshot / click / type_text)
       ↓
   yield tool_result 事件
       ↓
   结果回喂 LLM → 继续生成 (循环)
  ↓
 写入 assistant 消息 + tool_calls 持久化
  ↓
 yield done
```

## 三、Phase 0 — 基础架构（必须先行）

### 0.1 数据模型扩展

**文件**: `server/db/orm.py`

Persona 表新增字段:
- `tools_enabled: Boolean = True` — 工具调用总开关
- `rag_enabled: Boolean = True` — 知识库注入开关
- `sandbox_allow_network: Boolean = False` — 沙箱网络权限
- `terminal_allowed: Boolean = False` — 终端命令权限
- `browser_use_enabled: Boolean = False` — 浏览器操作权限

Message 表新增字段:
- `tool_calls: Text (JSON)` — LLM 返回的工具调用
- `tool_call_id: String(64)` — 工具调用 ID
- `tool_name: String(64)` — 工具名
- `tool_result: Text` — 工具执行结果

Conversation 表新增字段:
- `status: String(20) = "idle"` — idle/running/error

### 0.2 Function Calling 协议层

**文件**: `server/providers/protocols/openai_protocol.py`

`_build_payload` 新增:
```python
if kwargs.get("tools"):
    payload["tools"] = kwargs["tools"]
    payload["tool_choice"] = kwargs.get("tool_choice", "auto")
```

`_parse_sse_chunk` 新增解析 `delta.tool_calls`:
```python
if delta.get("tool_calls"):
    return StreamChunk(text="", tool_calls=delta["tool_calls"], ...)
```

`StreamChunk` 新增字段:
- `tool_calls: list[dict] | None = None`

### 0.3 工具调用循环

**文件**: `server/services/chat_service.py`

`stream_reply` 改造为工具调用循环:
1. 构建 `tools = list_tools_schema()`
2. 调用 provider，解析 tool_calls
3. 有 tool_calls → yield `{"type":"tool_call", ...}` → 执行 → yield `{"type":"tool_result", ...}`
4. 结果回喂 messages → 继续调用 provider（循环，最多 10 轮）
5. 无 tool_calls → 纯文本流 → 完成

### 0.4 SSE 事件协议扩展

新增事件类型:
- `{"type":"tool_call","id":"...","name":"...","arguments":{...}}`
- `{"type":"tool_result","id":"...","result":"...","success":true}`
- `{"type":"rag_context","sources":[{title,score,preview}]}`
- `{"type":"approval_required","tool":"...","arguments":{...}}`

### 0.5 前端 SSE 处理扩展

**文件**: `frontend/src/components/ChatPanel.tsx`

`sendMessage` 的 chunk 处理新增 tool_call/tool_result/rag_context 事件渲染。

**验收标准**:
- [ ] 数据模型迁移成功，旧数据兼容
- [ ] deepseek provider 能发起带 tools 的请求
- [ ] 能解析 tool_calls 响应
- [ ] 工具调用循环最多 10 轮，防死循环
- [ ] 前端能展示 tool_call 事件

## 七、Phase 4 — LanceDB 知识库（并行）

### 4.1 LanceDB 集成

**新建**: `server/services/knowledge_service.py`

```python
class KnowledgeService:
    """LanceDB 向量知识库 — RAGFlow 级功能"""
    
    def __init__(self):
        self._db = lancedb.connect(str(DATA_DIR / "knowledge.lance"))
        # 嵌入模型: 优先 OpenAI text-embedding-3-small, 无 key 时用本地 sentence-transformers
    
    async def ingest(self, source: str, content: str, metadata: dict) -> dict:
        """多模态文档摄入: 文本/图片/PDF/表格"""
    
    async def search(self, query: str, top_k: int = 5, filter: dict = None) -> list[dict]:
        """向量检索 + 元数据过滤"""
    
    async def build_graph(self) -> dict:
        """构建知识图谱: 实体抽取 + 关系链"""
```

### 4.2 文档解析器

**新建**: `server/services/doc_parser.py`
- PDF: `pypdf` 或 `pdfplumber`
- Word: `python-docx`
- Excel: `openpyxl`
- PPT: `python-pptx`
- 图片: OCR（优先系统 local-ocr-npu skill，降级 tesseract）
- Markdown: 直接读取

### 4.3 知识图谱

**新建**: `server/services/knowledge_graph.py`

## 十三、风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| LanceDB PyInstaller 打包兼容 | 安装失败 | 提前测试打包，备选 ChromaDB |
| Playwright 浏览器二进制过大 | 安装包膨胀 | 用系统已装浏览器，不下载 Chromium |
| 工具调用循环死循环 | API 费用爆炸 | 硬限制 10 轮 + token 计数限制 |
| 本地嵌入模型质量差 | 检索不准 | 无 key 时用 sentence-transformers + 降级提示 |
| PTY 在 Windows 兼容性 | 终端不可用 | pywinpty 优先 + mock 降级 |
